# Design System

## Principles

- Operational first: search, booking, payment, and admin actions must be clear before ornamental polish.
- No nested cards. Cards frame repeated items, forms, and status panels only.
- Compact admin tables for scanning and repeated action.
- Visible empty/error/success states.

## Tokens

- Background: `#f7f4ef`.
- Surface: `#ffffff`.
- Ink: `#1d2630`.
- Muted: `#667085`.
- Primary teal: `#126e82`.
- Accent rust: `#c65f2f`.
- Danger: `#b42318`.
- Success: `#157347`.
- Radius: 6-8px.
- Font: Inter/system sans-serif.

## Components

- Sticky topbar.
- Home hero with visual hotel background.
- Compact search panel.
- Search result rows and home room cards.
- Room detail with booking side panel.
- Status badges.
- Admin table pages.
- Mock payment panel.

## Responsive

- Main grid layouts collapse under 860px.
- Buttons and inputs use fixed minimum height and full-width controls where space is constrained.
- Text uses fixed sizes, not viewport-scaled type.
